   135→### PROMPT 02 — SUPABASE DATABASE SCHEMA
   136→> **Run this entire block in:** supabase.com → your project → SQL Editor → New Query → Run
   137→
   138→```sql
   139→-- ============================================================
   140→-- AI Career Navigator — Supabase Database Schema
   141→-- Run this full block once in the Supabase SQL Editor
   142→-- ============================================================
   143→
   144→CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
   145→
   146→-- User profiles (auto-created on signup via trigger below)
   147→CREATE TABLE public.profiles (
   148→  id          UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
   149→  full_name   TEXT,
   150→  email       TEXT UNIQUE NOT NULL,
   151→  created_at  TIMESTAMPTZ DEFAULT NOW()
   152→);
   153→
   154→-- Resume analysis results
   155→CREATE TABLE public.analyses (
   156→  id               UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
   157→  user_id          UUID REFERENCES public.profiles(id) ON DELETE SET NULL,
   158→  target_role      TEXT NOT NULL,
   159→  readiness_score  INTEGER CHECK (readiness_score BETWEEN 0 AND 100),
   160→  score_label      TEXT,
   161→  gaps             JSONB DEFAULT '[]',
   162→  strengths        JSONB DEFAULT '[]',
   163→  roadmap          JSONB DEFAULT '[]',
   164→  created_at       TIMESTAMPTZ DEFAULT NOW()
   165→);
   166→
   167→-- Anonymized skill trend tracking (no PII)
   168→CREATE TABLE public.skill_trends (
   169→  id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
   170→  target_role  TEXT NOT NULL,
   171→  skill_name   TEXT NOT NULL,
   172→  frequency    INTEGER DEFAULT 1,
   173→  last_seen    TIMESTAMPTZ DEFAULT NOW(),
   174→  UNIQUE (target_role, skill_name)
   175→);
   176→
   177→-- Row-level security
   178→ALTER TABLE public.profiles     ENABLE ROW LEVEL SECURITY;
   179→ALTER TABLE public.analyses     ENABLE ROW LEVEL SECURITY;
   180→ALTER TABLE public.skill_trends ENABLE ROW LEVEL SECURITY;
   181→
   182→-- Policies
   183→CREATE POLICY "Own profile readable"
   184→  ON public.profiles FOR SELECT USING (auth.uid() = id);
   185→
   186→CREATE POLICY "Own analyses readable"
   187→  ON public.analyses FOR SELECT USING (auth.uid() = user_id);
   188→
   189→CREATE POLICY "Own analyses insertable"
   190→  ON public.analyses FOR INSERT WITH CHECK (auth.uid() = user_id);
   191→
   192→CREATE POLICY "Skill trends public read"
   193→  ON public.skill_trends FOR SELECT USING (true);
   194→
   195→-- Auto-create profile on Supabase Auth signup
   196→CREATE OR REPLACE FUNCTION public.handle_new_user()
   197→RETURNS TRIGGER AS $$
   198→BEGIN
   199→  INSERT INTO public.profiles (id, email, full_name)
   200→  VALUES (
   201→    NEW.id,
   202→    NEW.email,
   203→    COALESCE(NEW.raw_user_meta_data->>'full_name', 'Anonymous')
   204→  );
   205→  RETURN NEW;
   206→END;
   207→$$ LANGUAGE plpgsql SECURITY DEFINER;
   208→
   209→CREATE OR REPLACE TRIGGER on_auth_user_created
   210→  AFTER INSERT ON auth.users
   211→  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
   212→```

   414→### PROMPT 06 — ANALYZE ROUTER
   415→
   416→```
   417→Using the master system context, generate the complete file:
   418→/career-navigator-backend/routers/analyze.py
   419→
   420→ENDPOINTS:
   421→
   422→POST /analyze/upload
   423→  Input: multipart/form-data
   424→    file: UploadFile  (PDF only, max 5MB)
   425→    target_role: str  (must be in VALID_ROLES)
   426→    user_id: Optional[str] = None
   427→  Process (in order):
   428→    1. Validate file content type is application/pdf; return HTTP 400 if not
   429→    2. Read file bytes; return HTTP 413 if > 5MB (5_242_880 bytes)
   430→    3. Call resume_parser.extract_text_from_pdf(bytes)
   431→    4. Call resume_parser.validate_resume(text); return HTTP 422 if invalid
   432→    5. Call resume_parser.is_resume(text); return HTTP 422 with message
   433→       "This does not appear to be a resume. Please upload your CV or resume."
   434→       if False
   435→    6. Call gemini_service.analyze_resume(text, target_role)
   436→    7. If result contains "error" key: return HTTP 502 with the error message
   437→    8. If user_id is provided:
   438→         a. Insert row into analyses table via supabase_admin
   439→         b. For each gap in result["gaps"], upsert into skill_trends:
   440→            on conflict (target_role, skill_name) increment frequency by 1
   441→         c. Add the saved analysis id to the result dict
   442→    9. Return result with HTTP 200
   443→
   444→GET /analyze/result/{analysis_id}
   445→  Fetch from analyses table by id using supabase_admin
   446→  Return 404 if not found, else return the full row as AnalysisResult
   447→
   448→GET /analyze/history/{user_id}
   449→  Return last 10 analyses for user_id, ordered by created_at DESC
   450→  Only return: id, target_role, readiness_score, score_label, created_at
   451→
   452→Include all imports. Import VALID_ROLES from models.schemas.
